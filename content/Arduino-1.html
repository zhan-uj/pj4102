Arduino-1

為了實現虛實整合，我們理想狀況是利用Arduino配合步進馬達來驅動實體機械轉盤計分器。由於不確定此方法的可行性，我們進行了第一版測試，先使用實體按鈕來觸發轉動，每按一次按鈕，馬達旋轉36度，並測試是否能成功帶動轉盤。

  1. 先將步進馬達與解碼器配合

 

      2.再將步進馬達解碼器與按鈕如下圖接至arduino開發板之對應腳位。

  

步進馬達第一版測試

#include <Stepper.h> 

Stepper stepper(100, 11, 10, 9, 8); // 初始化步進馬達，步數為100，使用引腳11, 10, 9, 8
#define but1 2 // 定義按鈕接在引腳2上

long currentPosition = 0; // 變數來記錄當前位置

void setup() 
{
  pinMode(but1, INPUT);  // 設置引腳2為輸入
  stepper.setSpeed(80);  // 設置馬達速度為100 RPM
  Serial.begin(115200);   // 初始化串口通信，波特率設置為115200
}

void loop() 
{
  // 檢查按鈕是否被按下
  if (digitalRead(but1) == HIGH)
  {
    stepper.step(50); // 正轉200步（36度）
    currentPosition += 200; // 更新當前位置
    delay(500);
  }

經過這次測試，確定了步進馬達帶動轉盤並正確旋轉36度此方法的可行性。並以此為基礎來實現虛擬場景與實體計分器之整合。
